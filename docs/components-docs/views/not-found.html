NotFound 404 页面

用于页面未找到时的展示，支持自定义标题、描述和返回按钮。

基础用法

最简单的 404 页面用法，使用默认样式和内容。

<solely-not-found></solely-not-found>

<!-- 全屏模式（页面级使用） -->
<solely-not-found></solely-not-found>

<!-- 嵌入模式（文档/组件内使用） -->
<solely-not-found embedded></solely-not-found>

自定义内容

使用属性自定义标题、描述和按钮文字。

<solely-not-found 
  title="404" 
  subtitle="页面走丢了"
  description="您访问的页面可能已被删除或暂时无法访问"
  back-text="返回首页"
  back-url="/">
</solely-not-found>

在路由中使用

配合 router-view 使用，作为 404 页面展示。

// 方式一：在 router-view 上指定
<router-view not-found="solely-not-found"></router-view>

// 方式二：在路由配置中指定
const routes: RouteConfig[] = [
  {
    path: '/',
    tagName: 'home-page',
    notFoundComponent: 'solely-not-found'
  }
];

API

属性

属性 说明 类型 默认值
title 主标题 string '404'
subtitle 副标题 string '抱歉，您访问的页面不存在'
description 描述文字 string '请检查网址是否正确，或点击下方按钮返回首页'
backText 返回按钮文字 string '返回首页'
backUrl 返回链接地址 string '/'
embedded 是否为嵌入模式（用于文档演示等场景） boolean false

方法

方法名 说明 参数 返回值
handleBack() 处理返回按钮点击，优先使用路由跳转 - void

类型定义

// 404 组件属性接口
interface NotFoundProps {
  title?: string;        // 主标题
  subtitle?: string;     // 副标题
  description?: string;  // 描述文字
  backText?: string;     // 返回按钮文字
  backUrl?: string;      // 返回链接地址
  embedded?: boolean;    // 是否为嵌入模式
}

路由配置

// RouteConfig 扩展属性
interface RouteConfig {
  // ... 其他属性
  
  /** 404 页面组件标签名 */
  notFoundComponent?: string;
}
