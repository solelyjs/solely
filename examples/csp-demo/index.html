Solely CSP 模式演示

展示 Solely 框架在严格 Content Security Policy 下的行为

CSP 已启用: script-src 'self'

当前 CSP 策略

Content-Security-Policy: default-src 'self'; script-src 'self'; // 不允许 'unsafe-eval' style-src 'self' 'unsafe-inline'; // Shadow DOM 内联样式需要 img-src 'self' data:;

两种编译模式对比

组件代码本身不会调用 new Function()，是框架内部的模板编译器在解析 {{ }}、:attr、onclick 等模板语法时需要动态生成函数。两种模式的区别在于这个编译过程发生的时机不同。

?solely 预编译模式

  • 构建时：Vite 插件将模板编译为 JS 函数
  • 生成的函数代码直接打包进产物
  • 运行时：直接执行已编译的函数
  • 不需要 new Function()，兼容 CSP

?raw 运行时编译模式

  • 运行时：浏览器中解析模板 AST
  • 遇到 {{ $data.count }} 等表达式时调用 new Function() 生成求值函数
  • CSP 阻止 new Function() → 表达式求值失败
  • 静态 DOM 结构可渲染，动态内容为空

实时对比

左右两个计数器功能完全相同，区别仅在于模板导入方式。?raw 模式下，组件的静态 HTML 骨架能渲染，但 {{ $data.count }} 等动态表达式因 CSP 阻止 new Function() 而无法求值。

?solely 预编译
构建时编译，运行时无需 new Function()

CSP 违规日志

如果页面中有代码尝试使用 eval() 或 new Function()，浏览器会阻止并在此显示违规报告。

暂无 CSP 违规，所有代码均在 CSP 策略允许范围内执行

如何在 CSP 环境中使用 Solely

// 1. 使用 ?solely 导入模板（预编译） import template from './component.html?solely' // 2. 在 Vite 插件中启用预编译 solelyVitePlugin({ precompile: true // 默认已开启 }) // 3. 配置 CSP（不需要 unsafe-eval） Content-Security-Policy: script-src 'self'; style-src 'self' 'unsafe-inline'; // 注意：Shadow DOM 内的样式需要 'unsafe-inline' // 如果不使用 Shadow DOM，可以移除 'unsafe-inline'
