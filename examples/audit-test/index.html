Solely 审查测试

针对 base-element-audit.md 中发现的问题进行验证（已同步 base-element.ts 最新实现）

核心验证

1. :prop 透传

验证 :message :count :active :items 动态绑定到子组件 $data

message: "{{ $data.parentMessage }}" | count: {{ $data.parentCount }} | active: {{ $data.parentActive }}

items: {{ this.json($data.parentItems) }}

2. reflect ↔ attribute 回环

验证 reflect: true 的 prop 修改后同步到 HTML attribute，已修复循环保护。

reflectValue: {{ $data.reflectValue }} | reflectLabel: "{{ $data.reflectLabel }}"

3. dispose() 显式销毁

验证 dispose() 后组件不可 reconnect，资源被正确清理。

状态: {{ $data.disposeStatus }}

Detach 后可 Reattach（状态保留）；Dispose 后尝试 Reattach 会抛错并被自动移除。

生命周期与边界

4. 兄弟组件挂载时序

Solely 使用 DocumentFragment 批量插入，兄弟节点在 mounted 时已同时存在于 DOM 中。

5. attributeChanged 回调

验证 attributeChanged(name, oldValue, newValue) 是否正确触发。

attrChangeValue: {{ $data.attrChangeValue }}

6. 静态属性解析（Upgrade Property）

不带 : 的属性值从字符串转换类型。非标准 JSON 会报错并回退到默认值，不再静默损坏。

标准 JSON 正常解析；含单引号的非标准 JSON 会 console.error 并回退到默认值。

7. array 类型 prop

动态绑定直接传递 JS 对象，不经过 JSON 序列化/反序列化。

items: {{ this.json($data.parentItems) }}

8. object + reflect 防御性验证

验证 type: 'object' 配合 reflect: true 时，循环保护是否生效。

点击"子改 config"后观察 attribute 同步情况，不应出现无限循环。

9. :$data 整体透传

验证整体替换子组件数据的可行性。

childData: {{ this.json($data.childData) }}

10. 对象/数组子属性透传

父组件子对象/子数组通过 :prop 传递到子组件，传递的是引用。

parentInfo: {{ this.json($data.parentInfo) }} | parentTags: {{ this.json($data.parentTags) }}
