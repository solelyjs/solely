针对 base-element-audit.md 中发现的问题进行验证（已同步 base-element.ts 最新实现）

1. :prop 透传测试

验证 :message :count :active :items 能否正确透传到子组件 $data

message: "{{ $data.parentMessage }}"

2. 静态属性设置（不带 : 前缀）

验证不带 : 的属性值需由 Upgrade Property 转换类型。2.2 已修复：移除了 replace(/'/g, '"')，非标准 JSON 会直接报错而非静默损坏。

① message="静态字符串" — 普通字符串

② count="5" — 数字类型需从字符串 "5" 转换

③ active — boolean 属性（存在即 true）

④ items='["x","y"]' — 标准JSON数组

⑤ message="it's a test" — 值含单引号（字符串类型，不经过 JSON 解析，正常）

⑥ items='["it's ok","hello"]' — 数组值含单引号（2.2 修复后 JSON.parse 会报错，使用默认值 []）

修复后行为：标准 JSON 正常解析；含单引号的非标准 JSON 会 console.error 并回退到默认值，而非静默损坏数据。

3. reflect ↔ attribute 回环测试

验证 reflect: true 的 prop 修改后是否同步到 HTML attribute。2.1 已修复：添加了 #reflectingAttrs 循环保护。

reflectValue: {{ $data.reflectValue }}

reflectLabel: "{{ $data.reflectLabel }}"

3-A. object + reflect 防御性验证

验证 type: 'object' 配合 reflect: true 时，#reflectingAttrs 能否防止 attribute ↔ state ↔ reflect 非幂等循环

点击"子改 config"后观察：attribute 是否正确同步，是否出现无限循环（页面卡死）。#reflectingAttrs 应在 reflect 写入 attribute 时跳过 attributeChangedCallback。

4. array 类型 prop 测试

验证 array 类型 prop 的动态绑定传递（动态绑定不经过 JSON 解析，不受 2.2 影响）

items: {{ this.json($data.parentItems) }}

动态 :items 直接赋值 JS 对象，不经过 JSON 序列化/反序列化，含单引号的数据完全正常。

5. :$data 整体透传测试

5-A. 对象/数组子属性透传

验证父组件 $data 的子对象/子数组通过 :prop 传递到子组件的 object/array 属性

父组件修改 $data.parentInfo（对象）和 $data.parentTags（数组）后，观察子组件的 info 和 items 是否同步更新。注意：对象/数组通过 :prop 传递的是引用，子组件直接修改会影响父组件。

8. dispose() 显式销毁测试

验证 dispose() 显式销毁后不可 reconnect（审查报告 2.8）
